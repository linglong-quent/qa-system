# AI Prompt 模板集

> 开发者在 AI 编码工具中使用这些模板，确保每次生成都携带规范约束。

---

## 模板 1：日常生成（完整上下文）

```
你负责为我们量化系统生成代码。请严格遵守项目根目录 CLAUDE.md 中的所有规则。

本次任务：
[描述具体需求]

要求：
1. 函数必须有完整类型注解
2. 数据处理用向量化，禁止行级循环
3. 异常处理精确，禁止 except: pass
4. 生成对应的 pytest 单元测试
5. 新增配置必须走 YAML，禁止硬编码
```

## 模板 2：数据处理函数生成

```
生成一个数据处理函数，输入是 [描述输入格式]，输出是 [描述输出格式]。

约束：
- 禁止 inplace=True
- 禁止 iterrows()
- NaN 处理策略必须显式声明 ffill / drop / 插值
- 返回 DataFrame 必须使用 .copy()
- 时区统一使用 UTC
- 如果涉及因子计算，必须 shift(1) 对齐时间戳
```

## 模板 3：回测代码生成

```
生成本次回测所需代码。

关键约束：
- 回测逻辑必须与实盘共用同一份代码
- 滑点和手续费参数化，禁止硬编码
- 测试集只能使用一次，禁止反复调整
- 回测结果必须包含 PSR（概率性夏普比）输出
- 随机种子必须固定
- 随机 K-Fold 禁止用于时序数据（必须用 CPCV 或时序分割）
```

## 模板 4：代码审查 / 重构

```
审查以下代码，找出违反 CLAUDE.md 规则的问题：
1. 是否混用时区？
2. 是否有前视偏差？
3. 是否有空 except？
4. 是否有性能瓶颈（循环、副本）？
5. 是否存在硬编码？

[粘贴代码]
```

## 模板 5：AI 生成特殊场景 — 修正代码

```
以下代码有合规问题。请修正：
- 类型注解不完整
- 使用了行级循环
- 没有处理 NaN
- 缺少单元测试

[粘贴代码]
```
