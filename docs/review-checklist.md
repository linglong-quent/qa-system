# Human Code Review Checklist

> AI 已经完成了 Pre-PR 合规审查，以下由人类 Reviewer 专注于 AI 无法判断的问题。
> 目标：10-15 分钟完成一个 PR，不需要逐行审合规。

---

## 1. 业务逻辑（AI 最容易在此出问题）

- [ ] **需求映射**：代码实现的逻辑是否符合 PR 描述的需求？不是"看起来对"就行
- [ ] **边界情况**：空数据、异常输入、市场休市时，这段代码会怎样？
- [ ] **数值正确性**：涉及金额/数量的计算是否有精度问题？（用 Decimal 了吗？）
- [ ] **时间处理**：时间戳比较的逻辑是否正确？有没有时区假设？
- [ ] **回测校准**：如果改动了回测逻辑，结果变化是否合理？

## 2. 数据流向（量化特有）

- [ ] **数据来源**：这个数据是从哪个层获取的？有没有绕过层直接访问？
- [ ] **前视偏差**：回测中的因子计算是否用了未来数据？（核对 `shift(1)` 使用）
- [ ] **数据清洗**：NaN 处理策略是否合理？缺失值有没有被无声填充？
- [ ] **缓存一致性**：如果使用了缓存，缓存失效策略是否合理？

## 3. AI 特别留意点

- [ ] **AI 幻觉**：代码中是否有 AI 编造的不存在的函数/API/库？
- [ ] **重复功能**：项目中是否已经有现成函数可以实现同样功能？
- [ ] **过度设计**：AI 有没有生成不必要的抽象层？（50 行能解决的问题写了 3 个类）
- [ ] **隐藏的硬编码**：测试文件、Notebook、注释、YAML 中有没有偷偷塞进去的密钥？
- [ ] **不必要的复杂度**：循环嵌套是否超过了 2 层？是否需要向量化？

## 4. 架构合规

- [ ] **跨层调用**：策略代码是否直接调用了数据库或交易所 API？
- [ ] **职责范围**：这个函数/类是否超出了它的职责？
- [ ] **配置分离**：新加的参数是硬编码的还是走配置文件的？
- [ ] **依赖声明**：新增的第三方库是否在 `pyproject.toml` 中声明？

## 5. 风险判断

- [ ] **最坏情况**：如果这个逻辑出错了，后果是什么？有没有防护？
- [ ] **幂等性**：如果重复执行这个代码（比如因重试），结果一致吗？
- [ ] **监控影响**：这段代码是否在关键路径上？有无增加延迟或内存的风险？

---

## 快速决策

| 发现 | 决策 |
|------|------|
| 业务逻辑错误 | ❌ 打回 + 说明具体问题 |
| 数据流向有前视偏差 | ❌ 打回，回测结果不可信 |
| AI 幻觉（不存在的 API） | ❌ 打回 + 修复后重新 Review |
| 性能隐患（大数据不向量化） | ❌ 打回 + 要求修改 |
| 缺少测试 | ⚠️ 要求补充，可先标记但不阻断 |
| 命名/风格问题 | ⚠️ 建议修改，不阻断 |
| 注释不够清晰 | ⚠️ 建议优化，不阻断 |
| 以上都没有 | ✅ Approved |
